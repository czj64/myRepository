1、项目简介

在当今数字化时代，医疗健康领域的信息化建设已经成为提升医疗服务质量和效率的关键手段。传统的医院就诊模式普遍存在挂号排队时间长、就诊流程繁琐、医疗资源分配不均等问题，患者往往需要花费大量时间在医院排队等候，而医院方面也面临着就诊秩序管理难度大、医生工作负荷不均衡等挑战。为了解决上述问题，提高医院运营效率，改善患者就医体验，我们开发了本套医院预约挂号系统。

本系统是一个基于B/S架构的Web应用，采用Java技术栈开发，后端使用Spring Boot 4.0.6框架，持久层采用Spring Data JPA搭配MySQL 8.0数据库，前端采用Thymeleaf模板引擎进行服务器端渲染。系统遵循经典的三层架构设计原则，将表现层、业务逻辑层和数据访问层进行解耦，使得系统具有良好的可维护性和可扩展性。

从用户角色角度划分，系统支持三种不同类型的用户：患者、医生和管理员，每种角色拥有不同的功能权限和操作界面。患者角色可以在系统中浏览科室和医生信息，根据自身病情选择合适的科室和医生进行在线预约挂号，预约时可以查看医生的职称、简介和坐诊时间，系统会自动校验预约时间是否在医生的坐诊时间内，同时检测同一时间段是否存在预约冲突，确保预约的合理性。患者还可以查看自己的预约记录列表，支持按预约状态筛选和按医生姓名搜索，对于尚未就诊的预约可以进行编辑修改或取消操作。就诊完成后，患者可以对医生进行评分和评论。此外，患者还可以在个人中心修改个人姓名、手机号和登录密码。

医生角色登录系统后可以查看专属的工作台页面，了解今日的预约概况和患者数量。医生可以管理自己的患者列表，按照待就诊、已完成、已取消等状态进行分类查看。在接诊过程中，医生可以为患者录入诊断信息、开具处方和添加备注，完成治疗后将预约状态更新为已完成。医生还可以查看个人坐诊时间安排，以及接收今日新患者的消息通知。

管理员角色拥有系统的最高管理权限，可以对科室进行新增、编辑、启用和停用操作，停用科室时会同时停用该科室下的所有医生。管理员可以新增医生并为其创建登录账号，也可以对医生进行启用和停用管理。在预约管理方面，管理员可以查看所有预约记录，支持按预约状态筛选和按患者姓名搜索，还可以修改预约的状态。在用户管理方面，管理员可以查看所有用户列表，按用户名或姓名进行模糊搜索，并查看患者、医生、管理员各角色的数量统计。系统的数据统计功能通过ECharts图表库以饼图和柱状图的形式直观展示预约数据的分布情况。

在技术实现方面，系统的登录认证使用HttpSession管理会话状态，三种角色通过独立的登录入口进入系统，后端通过检验session中用户的角色字段来控制权限访问。数据库设计方面，系统包含用户表、科室表、医生表、预约表和评价表共五张数据表，表之间通过外键建立关联，并使用ON DELETE CASCADE确保数据一致性。系统的一致性为主题。系统还提供了关键字搜索功能，在科室导航、用户管理、预约管理等多个页面支持模糊查询，方便用户快速找到目标信息。

通过本系统的实施，可以实现医院预约挂号的全程信息化管理，有效缩短患者排队等候时间，优化医院就诊流程，提升医疗资源的利用效率，为患者、医生和医院管理者三方提供便捷高效的服务平台。

2、项目模块

本系统包含以下五大功能模块：

（一）用户认证模块
- 患者注册：新用户填写用户名、密码、姓名、手机号完成注册
- 患者登录：输入用户名和密码进行身份验证
- 管理员登录：独立登录入口，验证角色为管理员
- 医生登录：独立登录入口，验证角色为医生
- 登出功能：销毁会话，根据来源角色跳转到对应登录页
- 个人信息管理：修改姓名、手机号和密码

（二）预约挂号模块
- 科室浏览：查看所有可用科室列表，支持按科室名称搜索
- 医生查看：按科室查看医生列表，展示职称和坐诊时间
- 医生详情：查看医生详细介绍和历史评价
- 创建预约：选择医生后填写预约日期、时间和患者信息
- 时间校验：自动校验预约时间不能是过去时间，必须在医生坐诊时间内
- 冲突检测：检查同一医生同一时间段是否已有预约（间隔至少60分钟）
- 我的预约：分页查看个人预约记录，支持按状态筛选和按医生姓名搜索
- 编辑预约：修改预约日期、时间和患者信息
- 取消预约：将预约状态更新为已取消

（三）就诊管理模块
- 医生工作台：展示今日预约概况和统计数据
- 患者管理：按待就诊、已完成、已取消等状态分类查看患者
- 就诊记录：查看患者就诊历史记录，分页显示
- 诊断录入：填写诊断内容、开具处方、添加备注
- 状态更新：就诊完成后将预约状态更新为已完成
- 排班查看：查看医生个人坐诊时间安排
- 消息通知：查看今日新患者通知

（四）后台管理模块
- 科室管理（完整CRUD）：
  - Create：新增科室（名称、描述）
  - Read：分页查看所有科室列表
  - Update：编辑科室信息，启用/停用科室
  - Delete：停用科室（软删除，同时停用该科室所有医生）
- 医生管理（完整CRUD）：
  - Create：新增医生并自动创建登录账号
  - Read：分页查看所有医生列表
  - Update：启用/停用医生
  - Delete：停用医生（软删除）
- 预约管理：
  - Read：分页查看所有预约，按状态和患者姓名筛选搜索
  - Update：修改预约状态
- 用户管理：
  - Read：分页查看所有用户列表，按用户名或姓名模糊搜索
  - 统计：展示患者、医生、管理员各角色数量
- 数据统计：
  - 概览卡片：展示医生总数、科室数量、预约总数、患者总数
  - 预约状态分布：饼图展示各状态预约占比
  - 今日预约情况：柱状图展示当日各状态预约数量

（五）评价模块
- 创建评价：患者对已完成就诊的医生进行评分（1-5分）和评论
- 查看评价：在医生详情页查看历史评价列表
- 平均评分：系统计算并展示医生的平均评分

3、同类型产品简介

（一）微信挂号（腾讯健康）
微信挂号是腾讯公司依托微信平台推出的在线医疗服务平台，用户可以通过微信中的"腾讯健康"小程序或公众号进行医院预约挂号。该平台覆盖了全国多家三甲医院，支持在线挂号、缴费、查看报告等功能。其优势在于用户基数庞大，使用便捷，无需单独安装App。但其不足之处在于不同医院的接入程度不一，部分医院仅提供基本信息查询，预约流程在不同医院之间存在差异。

（二）支付宝挂号（阿里健康）
支付宝挂号是阿里健康旗下的一项医疗服务，集成在支付宝App中。用户可以通过支付宝搜索医院名称进行预约挂号，支持在线支付挂号费用。该平台整合了蚂蚁金服的支付能力，提供先诊疗后付费等特色服务。阿里健康还推出了电子健康卡功能，方便患者跨院就诊。但其主要覆盖一二线城市的大型医院，对于基层医疗机构的覆盖相对不足。

（三）114预约挂号平台
114预约挂号平台是由中国电信推出的全国统一预约挂号服务平台，用户可以通过拨打114电话或访问网站进行预约挂号。该平台覆盖范围广泛，包括众多基层医院和社区卫生服务中心，特别适合不擅长使用智能手机的老年人群。但电话预约的效率相对较低，无法直观查看医生的详细介绍和患者的评价信息。

（四）平安好医生
平安好医生是中国平安集团旗下的在线医疗平台，提供在线问诊、预约挂号、健康管理等综合医疗服务。该平台的优势在于其强大的保险业务联动能力，用户可以在线咨询医生并获得用药建议。但平安好医生更侧重于在线问诊服务，在预约挂号方面的医院覆盖度和功能深度不及上述几个平台。

（五）各平台对比分析
与本系统相比，上述商业平台虽然具有用户基数大、覆盖医院多、资金实力雄厚等优势，但它们普遍存在系统功能过于通用化、难以针对特定医院进行深度定制的问题。本系统作为定制化开发的医院预约挂号系统，能够根据具体医院的管理流程和业务需求进行灵活配置，在科室管理、医生排班、预约规则设置等方面具有更高的灵活性。同时，本系统的数据统计和分析功能可以更好地满足医院管理层对运营数据的掌握需求，帮助医院优化资源配置，提升管理效率。
