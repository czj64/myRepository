二、项目建模（30分）

答：

1、数据库建模EER图

本系统包含以下5个实体模型：

（1）User（用户）
- 属性：id（主键）、username（用户名）、password（密码）、name（姓名）、phone（手机号）、role（角色）、created_at、updated_at
- 关联：一个用户可以有多个预约（1:N），一个用户可以有多条评价（1:N）

（2）Department（科室）
- 属性：id（主键）、name（科室名称）、description（描述）、active（是否启用）、created_at、updated_at
- 关联：一个科室可以有多个医生（1:N）

（3）Doctor（医生）
- 属性：id（主键）、name（姓名）、title（职称）、department_id（所属科室）、description（简介）、available_time（坐诊时间）、active（是否启用）、created_at、updated_at
- 关联：一个医生属于一个科室（N:1），一个医生可以有多个预约（1:N），一个医生可以有多条评价（1:N）

（4）Appointment（预约）
- 属性：id（主键）、user_id（预约用户）、doctor_id（预约医生）、appointment_date（日期）、appointment_time（时间）、status（状态）、patient_name（患者姓名）、patient_id_card（身份证号）、patient_phone（联系电话）、created_at、updated_at
- 关联：一个预约属于一个用户（N:1），一个预约属于一个医生（N:1），一个预约对应一条评价（1:1）

（5）Review（评价）
- 属性：id（主键）、appointment_id（预约ID）、user_id（用户ID）、doctor_id（医生ID）、rating（评分）、comment（评论）、created_at
- 关联：一条评价属于一个预约（N:1），一条评价属于一个用户（N:1），一条评价属于一个医生（N:1）

实体关联关系总结：
User（1）──〈（N）Appointment（N）〉──（1）Doctor（N）〉──（1）Department
User（1）──〈（N）Review（N）〉──（1）Doctor
Appointment（1）──〈（1）Review

2、项目应用建模

（1）架构图
系统采用经典的三层B/S架构：

表现层（Controller + Thymeleaf模板引擎）
    ↕ HTTP请求/响应
业务逻辑层（Service接口 + ServiceImpl实现类）
    ↕ DAO调用
数据访问层（Repository接口 + JPA/Hibernate）
    ↕ SQL
数据库（MySQL 8.0）

（2）接口标准
所有接口遵循RESTful风格，采用HTTP GET/POST方法：
- 查询操作：GET请求，参数通过URL传递
- 增删改操作：POST请求，参数通过表单提交
- 响应格式：服务器端渲染HTML页面（Thymeleaf）
- 登录状态：基于HttpSession管理
- 角色认证：通过session中的user.role字段校验权限


三、项目落地（40分）

答：

1、数据库（实体表、属性、关联）

系统共5张数据库表，均采用InnoDB引擎，字符集utf8mb4：

（1）user表
字段：id(BIGINT PK)、username(VARCHAR UNIQUE)、password(VARCHAR)、name(VARCHAR)、phone(VARCHAR UNIQUE)、role(INT)、created_at(TIMESTAMP)、updated_at(TIMESTAMP)
外键：无

（2）department表
字段：id(BIGINT PK)、name(VARCHAR UNIQUE)、description(VARCHAR)、active(BOOLEAN)、created_at(TIMESTAMP)、updated_at(TIMESTAMP)
外键：无

（3）doctor表
字段：id(BIGINT PK)、name(VARCHAR)、title(VARCHAR)、department_id(BIGINT)、description(VARCHAR)、available_time(VARCHAR)、active(BOOLEAN)、created_at(TIMESTAMP)、updated_at(TIMESTAMP)
外键：department_id → department(id) ON DELETE CASCADE

（4）appointment表
字段：id(BIGINT PK)、user_id(BIGINT)、doctor_id(BIGINT)、appointment_date(DATE)、appointment_time(TIME)、status(VARCHAR)、patient_name(VARCHAR)、patient_id_card(VARCHAR)、patient_phone(VARCHAR)、created_at(TIMESTAMP)、updated_at(TIMESTAMP)
外键：user_id → user(id) ON DELETE CASCADE，doctor_id → doctor(id) ON DELETE CASCADE

（5）review表
字段：id(BIGINT PK)、appointment_id(BIGINT)、user_id(BIGINT)、doctor_id(BIGINT)、rating(INT)、comment(VARCHAR)、created_at(TIMESTAMP)
外键：appointment_id → appointment(id) ON DELETE CASCADE，user_id → user(id) ON DELETE CASCADE，doctor_id → doctor(id) ON DELETE CASCADE

2、项目架构（模块、实体类、层级、CRUD完整性）

系统分为5大功能模块：
- 用户认证模块：登录、注册、登出、个人信息管理
- 预约挂号模块：科室浏览、医生查看、创建预约、编辑预约、取消预约、关键字搜索
- 就诊管理模块：医生工作台、患者管理、就诊记录录入、排班查看
- 后台管理模块：科室CRUD、医生CRUD、预约管理、用户管理、数据统计
- 评价模块：患者对医生评分评论

实体类5个（与数据库表一一对应）：
User.java、Department.java、Doctor.java、Appointment.java、Review.java

3层架构完整实现：
- Controller层：5个控制器，约45个接口
- Service层：5个Service接口 + 5个Impl实现类
- Repository层：5个JPA Repository接口

CRUD完整性：
每个实体均实现完整的增删改查操作，查询支持分页（Pageable）、排序、条件筛选和关键字模糊搜索。
