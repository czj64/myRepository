# 就诊管理模块 - 开发文档

## 1. 模块概述

就诊管理模块面向医生角色，提供工作台概览、患者管理、就诊记录录入、排班查看等功能，是医生处理日常诊疗工作的核心模块。

## 2. 功能列表

| 功能 | 说明 | 对应URL |
|------|------|---------|
| 医生工作台 | 今日预约概览 | GET /doctor/dashboard |
| 患者管理 | 按状态查看患者（待就诊/已完成/已取消） | GET /doctor/patients |
| 就诊记录 | 患者就诊历史记录（分页+搜索） | GET /doctor/patients模糊搜索） | GET /doctor/records |
| 编辑记录 | 录入诊断、处方、备注 | GET /doctor/record/edit |
| 保存记录 | 保存就诊记录并更新预约状态 | POST /doctor/doctor/record/save |
| 坐诊排班 | 查看医生个人坐诊时间 | GET /doctor/schedule |
| 消息通知 | 查看今日新患者通知 | GET /doctor/messages |
| 个人信息 | 查看和修改医生信息 | GET/POST /doctor/profile |

## 4. 核心业务逻辑

### 4.1 医生工作台

工作台展示医生今日的预约概况：
```
医生登录 → 获取医生信息 → 查询今日预约列表
    ↓
统计：待就诊数量、已完成数量、已取消数量
    ↓
展示：今日预约时间线 + 统计数据卡片
```

### 4.2 患者管理Tab切换

```java
// DoctorController.java - /doctor/patients
String statusFilter = ...; // 从请求参数获取
Page<Appointment> appointments;
if ("待就诊".equals(statusFilter)) {
    appointments = appointmentService.findByDoctorIdAndStatus(doctorId, "待就诊");
} else if ("已完成".equals(statusFilter)) {
    appointments = appointmentService.findByDoctorIdAndStatus(doctorId, "已完成");
} else {
    appointments = appointmentService.findByDoctorId(doctorId, pageable); // 全部
}
```

### 4.3 就诊记录录入

```java
// DoctorController.java - /doctor/record/save
① 获取预约ID → 查询预约记录
② 更新预约状态为"已完成"
③ 创建就诊记录（诊断内容、处方、备注）
④ 保存就诊记录到数据库
⑤ 重定向到患者列表页面
```

## 5. 数据表关联

就诊管理涉及以下表的联动：
- **appointment表**：医生通过 doctor_id 查询自己的患者预约记录
- **user表**：关联预约中的患者用户信息
- **doctor表**：用于医生登录验证和个人信息

## 6. 角色权限控制

// 医生端所有接口都需要校验角色
User user = (User) session.getAttribute("user");
