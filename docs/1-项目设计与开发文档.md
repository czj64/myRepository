# 医院预约挂号系统 - 项目设计与开发文档

## 1. 项目概述

### 1.1 项目简介
本项目为**医院预约挂号系统**，旨在为患者提供在线预约挂号服务，为医生提供就诊管理功能，为管理员提供系统管理功能。系统采用 B/S 架构，基于 Spring Boot 框架开发。

### 1.2 开发环境
- JDK 17
- Maven 3.8+
- IntelliJ IDEA
- MySQL 8.0+
- Spring Boot 4.0.6

### 1.3 技术栈

| 技术 | 用途 |
|------|------|
| Spring Boot 4.0.6 | 应用框架 |
| Spring MVC | Web层控制器 |
| Spring Data JPA | 持久层ORM |
| Thymeleaf | 服务器端模板引擎 |
| MySQL 8.0+ | 关系型数据库 |
| ECharts 5.4.3 | 数据可视化图表 |

### 1.4 运行环境

- 数据库：MySQL 3306，数据库名：rj2411260

## 2. 系统架构

### 2.1 整体架构
系统采用经典的三层架构：

```
表现层（Controller + Thymeleaf）
    ↓  HTTP 请求/响应
业务逻辑层（Service Interface + Impl）
    ↓  DAO 调用
数据访问层（Repository + JPA）
    ↓  SQL
数据库（MySQL）
```

## 4. 核心业务逻辑

### 4.1 预约流程

```
患者选择科室 → 选择医生 → 填写预约信息
    ↓
系统校验：
1. 时间不能是过去时间不能是过去时间
2. 时间必须在医生坐诊时间内
3. 同一医生同一时间段无冲突（间隔≥60分钟）
    ↓
校验通过 → 预约成功（状态：待就诊）
```

### 4.2 时间冲突检测
在 `AppointmentServiceImpl.hasTimeConflict()` 中实现，查询同一医生同一日期的所有预约，检查新预约时间与现有预约时间是否相差小于60分钟，如果冲突则拒绝预约。

### 4.3 坐诊时间校验
在 `AppointmentController.isWithinAvailableTime()` 中实现，解析医生坐诊时间配置（如"周一、周三、周五 上午8:00-12:00"），校验预约日期是否符合星期规则，时间是否在坐诊时间段内。

## 5. 安全设计

### 5.1 认证方式
- 使用 Session 管理登录状态
- 三个角色各自独立登录入口
- 未登录用户重定向到登录页

### 5.2 角色校验
- 管理员操作前检查 session 角色是否为 1
- 医生操作前检查角色是否为 2
- 患者只能查看和操作自己的预约数据

### 5.3 异常处理
- `GlobalExceptionHandler` 统一拦截异常
- 业务异常返回友好错误页面
- 系统异常返回 500 页面
