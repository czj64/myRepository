## 登录报错修复（23:22）
修复了 BCrypt 改造引入的两个兼容性Bug：

### Bug 1: save() 重复编码（P0）
- **问题**：UserServiceImpl.save() 自动编码密码，导致 updateProfile 等场景下重复编码
- **修复**：移除 save() 中的自动编码逻辑，改为在注册时显式调用 passwordEncoder.encode()
- **涉改**：UserServiceImpl.save()、HomeController.register()

### Bug 2: 数据库明文密码与BCrypt不兼容（P0）
- **问题**：数据库存的是明文"123456"，代码用 BCrypt.matches() 比对必然失败
- **修复**：新增 PasswordMigrationRunner，启动时自动将明文密码迁移为 BCrypt 哈希
- **逻辑**：检测 password 不以 "$2a$" 开头的记录，调用 encode() 后覆盖保存

## 预约时段解析失败修复（23:41）
- **问题**：前端 JS 正则解析中文坐诊时间文本失败，显示"无法解析医生坐诊时间"
- **根因**：Thymeleaf 模板中 `/*[[...]]*/` 语法输出到 JS 变量可能包含引号或编码问题
- **修复**：改为后端预解析 → 前端直接使用
  - `AppointmentController.java`：新增 `parseTimeRange()` 方法，`createAppointment()` 中解析 startTime/endTime 后传入 Model
  - `appointment-create.html`：JS 变量从 `doctorAvailableTime`（中文文本）改为 `timeStart`/`timeEnd`（已解析的时间），移除前端 parseTimeRange()

## 预约时段解析 v3 修复（23:50）
